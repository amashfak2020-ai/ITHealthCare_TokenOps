# Contributing to ITHealthCare_TokenOps

Thank you for your interest in contributing.

## How to Contribute

1. Fork the repository.
2. Create a feature branch.
3. Make small, focused changes.
4. Add or update documentation and tests where appropriate.
5. Submit a pull request with a clear summary of the change.

## Suggested Contribution Areas

- Token optimization strategies for healthcare workflows
- Prompt templates for clinical, payer, and support use cases
- Cost and latency evaluation utilities
- Compliance-oriented review checklists
- Documentation, diagrams, and usage examples

## Pull Request Guidelines

- Keep changes scoped and well described.
- Prefer readable, maintainable examples.
- Avoid including real patient data or sensitive information.
- Document assumptions for healthcare-specific workflows.
- Add tests for any executable logic.

## Code of Conduct

Please be respectful and constructive in all interactions.
